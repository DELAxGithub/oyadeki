# オヤデキ LINE インターフェース デザイン発注要件書

## 概要

**プロダクト名**: オヤデキ
**コンセプト**: 親御さんのデジタル困りごとを子どもと一緒に解決するLINEボット
**ターゲット**: 60代以上の親御さん（主要）、30〜40代の子ども世代（補助）

### UXコンセプト

**「普段は空気、必要な時だけ出てくる」**

- 親子グループで普通にLINEトークをしている
- 親が写真を送ると、オヤデキが自動で割り込んで反応
- AIが写真の内容を判定し、適切な対応を返す
  - 困りごと（スクショ）→ 解決手順を提示
  - メディア（TV・映画）→ 視聴記録として保存
  - 商品写真 → メルカリ出品文を生成

```
👴 親: Netflix でドラマ見た [写真]
     ↓
🤖 オヤデキ: （自動で割り込み）
   「ゴジラ-1.0 ですね！評価つけますか？」
   [⭐1] [⭐2] [⭐3] [⭐4] [⭐5]

👴 親: なんか変な画面出た [スクショ]
     ↓
🤖 オヤデキ: （自動で割り込み）
   「⚠️ これは詐欺の可能性があります！」
   【やること】1️⃣ ホーム押す 2️⃣ ...
```

## 機能の優先度

| 優先度 | 機能 | 説明 |
|--------|------|------|
| **メイン** | 契約台帳 | サービス・月額・IDの管理（パスワードは別管理） |
| **メイン** | 困りごと初期フォロー | スクショ→解決手順を3ステップで提示 |

| **サブ** | メルカリサポート | 商品写真→出品文生成 |

### 関連アプリ

| アプリ | プラットフォーム | 役割 |
|--------|-----------------|------|
| オヤデキ | LINE ボット | 親のデジタルサポート |
| シゴデキ | iOS アプリ | 家族共有リマインダー/TODO |

→ リッチメニューからシゴデキへディープリンクで連携

## 1. リッチメニュー

### 仕様
- **サイズ**: 2500px × 1686px（LINE公式仕様・2×2グリッド）
- **レイアウト**: 2行 × 2列
- **タップ領域**: 各ボタン 48px以上確保

### ボタン構成（2×2）

```
├──────────────┬──────────────┤
│   📑 契約台帳  │  📖 見たもの  │
├──────────────┼──────────────┤
│   💬 相談履歴  │  📲 シゴデキ  │
└──────────────┴──────────────┘
```

| 位置 | ラベル | 機能 |
|------|--------|------|
| 左上 | 契約台帳 | 登録済み契約の一覧表示 |
| 右上 | 見たもの | 視聴記録（メディアログ）一覧 |
| 左下 | 相談履歴 | 過去の困りごと相談を見返す |
| 右下 | シゴデキ | iOS アプリへディープリンク |

### 設計意図
- **設定ボタンなし**: 親に設定を意識させない
- **蓄積データを見返す用**: 普段は写真送信がメイン操作
- **シゴデキ連携**: 家族アプリエコシステムへの導線

### デザイン方針
- **視認性重視**: 大きめアイコン + 短いラベル
- **コントラスト**: 4.5:1以上（アクセシビリティ）
- **トーン**: 温かみのある親しみやすさ（堅すぎない）

### カラーバリエーション（3種類）
1. **レッド系** - 活発・元気なイメージ
2. **ブラック系** - シンプル・モダン
3. **ゴールド系** - 上品・信頼感

## 2. Flex Message テンプレート

### 共通デザインルール

#### カラーパレット
```
プライマリ（成功・確認）: #1DB446（LINE Green）
セカンダリ（補足）      : #666666
警告（注意喚起）        : #E65100（オレンジ）
エラー（危険）          : #CC0000（赤）
背景ソフト              : #F0FFF4（薄緑）, #FFF8E1（薄黄）, #FFE0E0（薄赤）
```

#### タイポグラフィ
- **見出し**: Bold, 16-18px
- **本文**: Regular, 14px, 行間1.5
- **補足**: 12px, グレー（#888888）

#### ボタン
- **高さ**: 48px以上（タップしやすさ）
- **角丸**: 8px
- **スタイル**: Primary（塗り）/ Secondary（枠線）/ Link（テキストのみ）

### 2.1 救急箱回答メッセージ

**用途**: スマホの画面写真を送ると、解決手順を返す
**トリガー**: 写真送信 → AI判定「困りごと」

```
┌─────────────────────────────────┐
│ ⚠️ 警告バナー（危険時のみ）      │  背景: #FFE0E0
│ 「これは詐欺の可能性が...」      │  文字: #CC0000
├─────────────────────────────────┤
│ 📱 状況説明                      │
│ 「〇〇の画面が表示されています」 │
├─────────────────────────────────┤
│ 【やること】                     │
│ 1️⃣ ホームボタンを押す           │
│ 2️⃣ ブラウザを閉じる             │
│ 3️⃣ 設定を確認                   │
├─────────────────────────────────┤
│ 📞 難しければお子さんに電話を    │  ※複雑時のみ表示
├─────────────────────────────────┤
│ [わかった！] [電話で聞く]        │  ボタン2つ
│ [📑 これを台帳に登録]            │  ※契約画面時のみ
└─────────────────────────────────┘
```

**ポイント**:
- 手順は最大3つまで（スクロール防止）
- 各手順は絵文字で視覚的に番号付け
- 警告バナーは目立つ赤系で危機感を演出

## 3. LIFF 設定画面

### 設定の導線

**親には触らせない設計**

1. **初回（友だち追加時）**
   - 子どもが親のスマホで初期設定
   ```
   🤖: オヤデキへようこそ！
       まず設定をお願いします
       [設定を開く] ← LIFF
   ```

2. **あとから（コマンド）**
   - 必要な時だけ「設定」と送信
   ```
   👦 子: 設定
   🤖: 設定画面はこちら
       [設定を開く] ← LIFF
   ```

### 画面構成

```
┌─────────────────────────────────┐
```

## 7. 技術的な制約

### LINE Flex Message
- 最大バブル数: 12個（カルーセル）
- 最大ボタン数: 4個/バブル
- 画像は外部URL必須（Base64不可）
- フォント指定不可（システムフォント使用）

### アクセシビリティ
- コントラスト比: 4.5:1以上
- タップ領域: 48×48px以上
- フォントサイズ: 最小14px
